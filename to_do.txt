- PROJECT AI CHAT IMPROVEMENT

I have an Odyssey project management web app (repo: khicks1724/odyssey).
The app has a per-project AI chat panel that currently receives GitHub and GitLab
repository context via commit history, README files, and a limited number of
code files. I need you to expand this so the AI chat window has access to the
FULL source code of every linked repository — GitHub, GitLab, and eventually
Microsoft 365 — not just a surface-level summary.

## Current Behavior (what exists today)

The server fetches:
- Recent commit messages and diffs
- README file content
- A small number of code files (appears to be a fixed limit)

The AI system prompt is assembled in the server and passed to the AI provider
(Anthropic Claude, OpenAI GPT-4o, or Google Gemini depending on user selection).

## Goal

When a user sends a message in the project AI chat panel, the system prompt
sent to the AI should include:

1. FULL recursive file tree for every linked GitHub and GitLab repository
2. FULL file contents for every non-binary file (respecting a token budget)
3. A token-aware truncation strategy so the most relevant files are prioritized
   when the total content exceeds context limits
4. Caching of fetched repo content (TTL ~5 minutes) so repeated messages in
   the same session don't hammer the API rate limits
5. Support for the existing GitHub integration (github_repo field) and all
   linked GitLab repos (gitlab_repos array on the project)
6. Graceful fallback: if a repo is unreachable or rate-limited, include what
   is available and note what was skipped

## Technical Context

- Backend: Fastify 5, TypeScript, Node.js
- The AI context assembly likely lives in the server — look for files that
  build the system prompt or context string, probably something like
  server/src/ai.ts, server/src/routes/chat.ts, server/src/lib/github.ts,
  server/src/lib/gitlab.ts, or similar
- GitHub API: use the Git Trees API endpoint with recursive=1 to get the full
  file tree in one call, then fetch individual blobs for file contents
  GET /repos/{owner}/{repo}/git/trees/{sha}?recursive=1
  GET /repos/{owner}/{repo}/contents/{path}
- GitLab API: use the repository tree endpoint with recursive=true
  GET /projects/{id}/repository/tree?recursive=true&per_page=100
  GET /projects/{id}/repository/files/{file_path}/raw?ref=main
- Tokens: estimate ~4 chars per token; target staying under 180k tokens for
  the repo content block to leave room for conversation history and response
- Binary file exclusions: skip .png, .jpg, .jpeg, .gif, .ico, .pdf, .zip,
  .tar, .gz, .bin, .onnx, .pt, .weights, .h264, .mp4, .so, .dylib, .exe
- Large file threshold: skip individual files over 100KB
- Priority ordering when truncating (highest to lowest):
    1. Files matching the user's current message keywords
    2. Entry point files (main.py, main.ts, app.py, index.ts, server.ts)
    3. Config files (config.py, config.ts, package.json, requirements.txt)
    4. Source files (.py, .ts, .tsx, .js) under 20KB
    5. Everything else
